## 🎯 Core Features

### ✨ Three core competencies

#### 1️⃣ Hypotheses Tree

**Goal**: Structured analysis framework

```
business issues
  → STEP 1: Define problem boundaries (Is & Isn't)
  → STEP 2: Issue Tree (MECE dismantles 2-3 layers)
  → STEP 3: Hypothesis Tree (hypothesis driven + fast search)
```

**Output**:
- Clear problem definition
- Structured problem tree
- Testable hypotheses and storylines

---

#### 2️⃣ Dummy Pages (page design)
**Goal**: McKinsey style page planning

```
Hypothesis Tree
  → STEP 4: Determine the argument method and information sources
  → STEP 5: Design McKinsey Style Dummy Pages
```

**Output**:
- Argument framework for each page
- Detailed page layout design
- Precise list of data requirements

---

#### 3️⃣ PPT Generation (ultimate version generation)
**Goal**: Perfect McKinsey style PPT

```
Dummy Pages + Data
  → STEP 6: web_search data collection (15-30 times)
  → STEP 7: Call mckinsey-ppt-v4 to generate
  → STEP 8: Iterative optimization (2-4 rounds)
```

**Output**:
- First version of McKinsey style PPT (pages 15-25)
- Ultimate version PPT (95 points quality)

## 🏆Quality Assurance

### Content quality
- ✅ Issue Tree complies with MECE principles
- ✅ Assumptions have logical support and data verification
- ✅ Data comes from multiple reliable sources
- ✅ Storyline is coherent and has sufficient arguments

### PPT quality (McKinsey standard)
- ✅ Dark blue background + white text (contrast)
- ✅ Use right-angled rectangles for large text boxes (professional feel)
- ✅ Borderless by default (minimalism)
- ✅ Argument-style title (each page is a complete argument)
- ✅ High information density but not crowded
